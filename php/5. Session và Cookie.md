## Session và Cookie trong PHP

Session và Cookie đều dùng để **ghi nhớ thông tin giữa nhiều request**.

HTTP vốn là **stateless**: mỗi lần trình duyệt gửi request lên server, server không tự nhớ người dùng đó là ai. Vì vậy, PHP cần session hoặc cookie để duy trì trạng thái như đăng nhập, giỏ hàng, theme, ngôn ngữ, thông báo tạm thời.

```text
HTTP request 1 → Server xử lý → Kết thúc
HTTP request 2 → Server xử lý → Không tự nhớ request trước
```

Session và Cookie giải quyết vấn đề này.

---

## 1. Khác nhau giữa Session và Cookie

```text
Session
├── Dữ liệu lưu chủ yếu trên server
├── Trình duyệt chỉ giữ session id
├── An toàn hơn cookie
├── Thường dùng cho đăng nhập
└── Ví dụ: user_id, role, cart, flash message

Cookie
├── Dữ liệu lưu trên trình duyệt
├── Người dùng có thể xem hoặc sửa
├── Có thời hạn sống rõ ràng
├── Dùng cho dữ liệu ít nhạy cảm
└── Ví dụ: theme, language, remember token
```

Ví dụ dễ hiểu:

```text
Session giống tủ đồ ở server
├── Server giữ đồ thật
└── Browser chỉ giữ vé gửi đồ

Cookie giống tờ giấy lưu trong browser
├── Browser giữ trực tiếp dữ liệu
└── Server đọc lại khi browser gửi request
```

# A. Session

## 2. `session_start`

`session_start()` dùng để ==bắt đầu hoặc tiếp tục một session.==

Muốn dùng `$_SESSION`, gần như luôn phải gọi:

```php
<?php
session_start();
?>
```

Nên đặt `session_start()` ở đầu file, trước khi xuất HTML.

Đúng:

```php
<?php
session_start();

$_SESSION["username"] = "tai";
?>

<h1>Xin chào</h1>
```

Không nên:

```php
<h1>Xin chào</h1>

<?php
session_start();
?>
```

Lý do: session thường cần gửi header HTTP. Nếu bạn đã output HTML trước đó, PHP có thể báo lỗi kiểu:

```text
headers already sent
```

---

## 3. `$_SESSION`

`$_SESSION` là mảng dùng để lưu dữ liệu phiên làm việc của người dùng.

Ví dụ lưu thông tin sau khi đăng nhập:

```php
<?php
session_start();

$_SESSION["user_id"] = 15;
$_SESSION["username"] = "tai";
$_SESSION["role"] = "admin";

echo "Đã lưu session";
?>
```

Ở trang khác, bạn có thể đọc lại:

```php
<?php
session_start();

echo $_SESSION["username"];
?>
```

Kết quả:

```text
tai
```

Session thường dùng cho:

```text
$_SESSION
├── Lưu trạng thái đăng nhập
├── Lưu user_id
├── Lưu role của user
├── Lưu giỏ hàng tạm thời
├── Lưu flash message
└── Lưu CSRF token
```

---

## 4. Kiểm tra người dùng đã đăng nhập chưa

Ví dụ file `dashboard.php`:

```php
<?php
session_start();

if (!isset($_SESSION["user_id"])) {
    header("Location: login.php");
    exit;
}

echo "Chào mừng user #" . $_SESSION["user_id"];
?>
```

Ý nghĩa:

```text
Nếu có $_SESSION["user_id"]
├── Người dùng đã đăng nhập
└── Cho vào dashboard

Nếu không có $_SESSION["user_id"]
├── Người dùng chưa đăng nhập
└── Chuyển về login.php
```

---

## 5. `session_destroy`

`session_destroy()` dùng để hủy session hiện tại, ==thường dùng khi đăng xuất==.

Ví dụ file `logout.php`:

```php
<?php
session_start();

session_destroy();

header("Location: login.php");
exit;
?>
```

Cẩn thận: `session_destroy()` hủy session trên server, nhưng dữ liệu trong `$_SESSION` ở request hiện tại có thể vẫn còn trong bộ nhớ tạm. Cách logout sạch hơn:

```php
<?php
session_start();

$_SESSION = [];

session_destroy();

header("Location: login.php");
exit;
?>
```

Nếu muốn xóa luôn session cookie:

```php
<?php
session_start();

$_SESSION = [];

if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();

    setcookie(
        session_name(),
        "",
        time() - 42000,
        $params["path"],
        $params["domain"],
        $params["secure"],
        $params["httponly"]
    );
}

session_destroy();

header("Location: login.php");
exit;
?>
```

Đoạn trên là cách logout kỹ hơn trong ứng dụng thật.

# B. Cookie

## 6. `setcookie`

`setcookie()` dùng để gửi cookie từ server về trình duyệt.

Cú pháp cơ bản:

```php
<?php
setcookie("theme", "dark", time() + 3600);
?>
```

Ý nghĩa:

```text
theme
├── Tên cookie

dark
├── Giá trị cookie

time() + 3600
└── Cookie sống 3600 giây, tức 1 giờ
```

Ở request tiếp theo, trình duyệt sẽ gửi cookie này lại cho server.

Ví dụ dùng cookie để lưu theme:

```php
<?php
setcookie("theme", "dark", time() + 86400);
?>
```

Cookie này tồn tại 1 ngày.

---

## 7. `$_COOKIE`

`$_COOKIE` dùng để đọc cookie do trình duyệt gửi lên server.

```php
<?php
if (isset($_COOKIE["theme"])) {
    echo "Theme hiện tại: " . $_COOKIE["theme"];
} else {
    echo "Chưa có theme";
}
?>
```

Lưu ý quan trọng: sau khi gọi `setcookie()`, cookie chưa xuất hiện ngay trong `$_COOKIE` ở cùng request hiện tại. Nó thường chỉ có ở request tiếp theo.

Ví dụ:

```php
<?php
setcookie("theme", "dark", time() + 3600);

echo $_COOKIE["theme"] ?? "Chưa đọc được trong request hiện tại";
?>
```

Muốn thấy cookie, bạn reload trang hoặc gửi request mới.

---

## 8. Xóa cookie

Muốn xóa cookie, đặt thời gian hết hạn trong quá khứ:

```php
<?php
setcookie("theme", "", time() - 3600);
?>
```

Ví dụ:

```php
<?php
setcookie("language", "", time() - 3600);

echo "Đã xóa cookie language";
?>
```

# C. Login state

## 9. `login state` là gì?

`login state` là trạng thái cho biết người dùng hiện tại **đã đăng nhập hay chưa**.

Trong PHP thuần, ==cách phổ biến nhất là dùng session:==

```text
Login state
├── Chưa đăng nhập
│   └── Không có $_SESSION["user_id"]
│
└── Đã đăng nhập
    └── Có $_SESSION["user_id"]
```

>--> tức là sau khi đăng nhập thì trình duyệt sẽ tạo session lưu giá trị tạm user_id
>            --> lúc này kiểm tra bằng cách lấy user_id trong session xem có được ko   
>            $_SESSION["user_id"]		
>     --> lúc logout thì đơn giản là xóa luôn cái dữ liệu phiên session của người dùng


Sau khi user nhập đúng email/password, ta lưu `user_id` vào session.

---

## 10. Ví dụ login tối giản

File `login.php`:

```php
<?php
session_start();

$error = "";

if ($_SERVER["REQUEST_METHOD"] === "POST") {
    $email = $_POST["email"] ?? "";
    $password = $_POST["password"] ?? "";

    if ($email === "admin@example.com" && $password === "123456") {
        $_SESSION["user_id"] = 1;
        $_SESSION["email"] = $email;

        header("Location: dashboard.php");
        exit;
    }

    $error = "Sai email hoặc mật khẩu";
}
?>

<form method="POST">
    <input type="email" name="email" placeholder="Email">
    <input type="password" name="password" placeholder="Mật khẩu">
    <button type="submit">Đăng nhập</button>
</form>

<p><?= htmlspecialchars($error) ?></p>
```

File `dashboard.php`:

```php
<?php
session_start();

if (!isset($_SESSION["user_id"])) {
    header("Location: login.php");
    exit;
}
?>

<h1>Dashboard</h1>
<p>Xin chào <?= htmlspecialchars($_SESSION["email"]) ?></p>

<a href="logout.php">Đăng xuất</a>
```

File `logout.php`:

```php
<?php
session_start();

$_SESSION = [];
session_destroy();

header("Location: login.php");
exit;
?>
```

Luồng hoạt động:

```text
Người dùng nhập email/password
├── PHP kiểm tra thông tin
│   ├── Sai
│   │   └── Báo lỗi
│   └── Đúng
│       ├── Lưu $_SESSION["user_id"]
│       └── Chuyển sang dashboard.php
│
Dashboard
├── Nếu có $_SESSION["user_id"]
│   └── Cho xem nội dung
└── Nếu không có
    └── Đẩy về login.php
```

# D. Session + Cookie phối hợp thế nào?

Khi gọi `session_start()`, PHP thường tạo một cookie tên là `PHPSESSID`.

```text
Browser
└── Cookie: PHPSESSID=abc123

Server
└── Session data của abc123
    ├── user_id = 1
    ├── email = admin@example.com
    └── role = admin
```

Trình duyệt không cần giữ toàn bộ dữ liệu đăng nhập. Nó chỉ giữ `PHPSESSID`. Server dùng ID đó để tìm dữ liệu session tương ứng.

Đây là lý do session an toàn hơn cookie thông thường: dữ liệu chính nằm trên server.

# E. Cookie “Remember me”

==Session thường mất khi người dùng đóng trình duyệt hoặc session hết hạn.== Nếu muốn “ghi nhớ đăng nhập”, có thể dùng cookie, nhưng không được lưu password trong cookie.
![[Pasted image 20260509205059.png]]

Không an toàn:

```php
setcookie("password", "123456", time() + 86400 * 30);
```

An toàn hơn về mặt thiết kế:

```text
Remember me
├── Tạo token ngẫu nhiên
├── Lưu token đã hash vào database
├── Gửi token thô vào cookie
├── Request sau kiểm tra token
└── Nếu hợp lệ thì tạo lại session
```

Ví dụ minh họa đơn giản:

```php
<?php
$token = bin2hex(random_bytes(32));

setcookie("remember_token", $token, time() + 86400 * 30, "/", "", false, true);
?>
```

Tham số cuối `true` là `HttpOnly`, giúp JavaScript không đọc được cookie đó.

# F. Bảo mật cần nhớ

==Session và cookie liên quan trực tiếp đến đăng nhập,== nên phải cẩn thận.

```text
Quy tắc bảo mật
├── Luôn gọi session_start() trước khi dùng $_SESSION
├── Không lưu password trong session hoặc cookie
├── Không lưu dữ liệu nhạy cảm trực tiếp trong cookie
├── Dùng password_hash() khi lưu mật khẩu
├── Dùng password_verify() khi kiểm tra mật khẩu
├── Regenerate session id sau khi login
├── Dùng HttpOnly cookie
├── Dùng Secure cookie nếu chạy HTTPS
├── Validate dữ liệu từ $_COOKIE
└── Logout thì xóa session rõ ràng
```

Sau khi đăng nhập thành công, nên gọi:

```php
<?php
session_regenerate_id(true);
?>
```

Ví dụ:

```php
<?php
session_start();

if ($loginSuccess) {
    session_regenerate_id(true);

    $_SESSION["user_id"] = $user["id"];
}
?>
```

Lệnh này giúp giảm nguy cơ session fixation.

# G. Bảng nhớ nhanh

```text
Session và Cookie
├── session_start()
│   └── Bắt đầu hoặc tiếp tục session
│
├── $_SESSION
│   └── Lưu dữ liệu phiên trên server
│
├── session_destroy()
│   └── Hủy session, thường dùng khi logout
│
├── setcookie()
│   └── Tạo, cập nhật hoặc xóa cookie
│
├── $_COOKIE
│   └── Đọc cookie từ browser gửi lên
│
└── login state
    ├── Đã đăng nhập
    │   └── Có $_SESSION["user_id"]
    └── Chưa đăng nhập
        └── Không có $_SESSION["user_id"]
```

## Công thức thực tế nên nhớ

```php
<?php
session_start();

if (!isset($_SESSION["user_id"])) {
    header("Location: login.php");
    exit;
}
?>
```

Đây là mẫu kiểm tra đăng nhập cơ bản nhất trong PHP thuần.
